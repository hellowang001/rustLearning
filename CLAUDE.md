## 项目概述

这是一个 Solana/Anchor 学习项目。`learn/` 目录包含一个 Anchor 工作区，用于实验各种 Solana 概念（rent、账户、PDA 等）。

## 学习目标

- 掌握 Solana 链上程序（智能合约）的开发模式
- 熟练使用 Anchor 框架编写 Rust 合约
- 理解 Solana 核心概念：账户模型、PDA、rent、CPI 等
- 学习 SPL Token 程序的使用与 CPI 调用
- 通过 TypeScript 测试验证合约行为

## Go 客户端代码生成器（main.go）

`main.go` 是一个 IDL → Go 代码生成器，读取 Anchor 编译产出的 IDL JSON，自动生成 Go 客户端代码。

### 用法

```bash
# 在仓库根目录执行
go run main.go <idl.json> <output.go> <program-id>

# 示例：为 bank 包生成客户端
go run main.go learn/target/idl/keypair_vs_pda.json bank/bank.go 8sm8zjwUwhvBLxhv1RHd64fQvdAasxoE3j57NrqSxJdr
```

> 包名由输出文件的目录名自动推断（如 `bank/bank.go` → `package bank`）。

### 生成内容

| 内容 | 说明 |
|------|------|
| 账户结构体 | 从 IDL `types[]` 读取字段，含 `UnmarshalWithDecoder` |
| `GetXxx()` 查询函数 | 直接读取链上账户数据，**跳过 8 字节 Anchor discriminator** |
| 指令构建器 `NewXxxInstruction()` | discriminator 直接从 IDL 读取，无需手动计算 |
| `ErrorCode` 错误映射 | `map[uint32]string` |
| `SendTransaction()` | 构建、签名、发送交易的辅助函数 |

### 注意事项

- **合约更新后**须重新运行生成器，IDL 文件路径：`learn/target/idl/keypair_vs_pda.json`
- **PDA 推导函数**（如 `FindUserAccountPDA`）不会自动生成，需手动维护在测试文件中
- **只读查询**（获取余额等）直接调用 `GetXxx()` 即可，无需发送交易；链上 `get_balance` 指令的返回值客户端无法直接获取，只能通过解析交易日志读取
- 生成的文件开头有 `// Code generated by idl2go. DO NOT EDIT.` 注释，不要手动修改

## 学习笔记

### SOL 转移（Transfer SOL）

在 Anchor 合约中转移原生 SOL，使用 `system_program::transfer` CPI。

#### Rust 合约

```rust
use anchor_lang::system_program;

pub fn transfer_sol(ctx: Context<TransferSol>, amount: u64) -> Result<()> {
    let cpi_ctx = CpiContext::new(
        ctx.accounts.system_program.to_account_info(),
        system_program::Transfer {
            from: ctx.accounts.from.to_account_info(),
            to:   ctx.accounts.to.to_account_info(),
        },
    );
    system_program::transfer(cpi_ctx, amount)?;
    Ok(())
}

#[derive(Accounts)]
pub struct TransferSol<'info> {
    #[account(mut)]
    pub from: Signer<'info>,       // 付款方，必须是签名者
    #[account(mut)]
    pub to: SystemAccount<'info>,  // 收款方（普通钱包账户）
    pub system_program: Program<'info, System>,
}
```

**要点：**
- `from` 必须是 `Signer`，否则 System Program 拒绝扣款
- `from` 和 `to` 都要加 `#[account(mut)]`，因为余额会变化
- `amount` 单位是 **lamports**（1 SOL = 1_000_000_000 lamports）
- 收款方用 `SystemAccount` 类型（普通钱包），若是合约 PDA 账户则用 `Account<'info, T>`

#### 从 PDA 转出 SOL

PDA 没有私钥，不能作为普通 Signer，需要用 `CpiContext::new_with_signer`：

```rust
let seeds = &[b"my_pda", &[ctx.bumps.pda]];
let signer_seeds = &[&seeds[..]];

let cpi_ctx = CpiContext::new_with_signer(
    ctx.accounts.system_program.to_account_info(),
    system_program::Transfer {
        from: ctx.accounts.pda.to_account_info(),
        to:   ctx.accounts.to.to_account_info(),
    },
    signer_seeds,
);
system_program::transfer(cpi_ctx, amount)?;
```
