⏺ SPL Token vs ERC20 对比

  相同点

  ┌──────────┬────────────────────────┬────────────────────────┐
  │   方面   │       SPL Token        │         ERC20          │
  ├──────────┼────────────────────────┼────────────────────────┤
  │ 目标     │ 同质化代币标准         │ 同质化代币标准         │
  ├──────────┼────────────────────────┼────────────────────────┤
  │ 核心功能 │ transfer / mint / burn │ transfer / mint / burn │
  ├──────────┼────────────────────────┼────────────────────────┤
  │ 元数据   │ name, symbol, decimals │ name, symbol, decimals │
  ├──────────┼────────────────────────┼────────────────────────┤
  │ 授权机制 │ delegate (approve)     │ approve + allowance    │
  └──────────┴────────────────────────┴────────────────────────┘

  ---
  核心差异

  1. 账户模型 vs 合约模型

  ERC20 — 合约内维护一张 mapping 表：

  // 所有余额都在同一个合约的 storage 里
  mapping(address => uint256) public balances;

  SPL Token — 每个持币地址有独立的 Token Account（链上账户）：

  Mint Account         ← 代币定义（supply, decimals, mint_authority）
      ↓
  Token Account (用户A) ← 存放用户 A 持有的该代币余额
  Token Account (用户B) ← 存放用户 B 持有的该代币余额

  ▎ 这是最根本的差异：SPL Token 的状态分散在多个独立账户中，而不是合约内部。

  ---
  2. 程序是共享的

  ┌──────────┬──────────────────────────┬───────────────────────────────────┐
  │          │          ERC20           │             SPL Token             │
  ├──────────┼──────────────────────────┼───────────────────────────────────┤
  │ 逻辑在哪 │ 每个代币自己部署一个合约 │ 所有代币共用同一个 spl-token 程序 │
  ├──────────┼──────────────────────────┼───────────────────────────────────┤
  │ 地址含义 │ 合约地址 = 代币标识      │ Mint 账户地址 = 代币标识          │
  └──────────┴──────────────────────────┴───────────────────────────────────┘

  ERC20 每发行一个新代币 → 部署一个新合约。
  SPL Token 每发行一个新代币 → 只需创建一个新的 Mint Account，程序逻辑复用。

  ---
  3. Token Account (ATA)

  用户要持有某个 SPL Token，必须先创建对应的 Associated Token Account (ATA)：

  用户钱包地址 + Mint地址  → 确定性派生出 ATA 地址 (PDA)

  - ERC20：转账前不需要任何准备，直接 transfer(to, amount)
  - SPL Token：接收方必须已存在 ATA，否则交易失败（或由发送方帮忙创建，但要付
  rent）

  ---
  4. Rent（租金）

  SPL Token 引入了链上存储的 rent 机制：

  - 创建 Mint Account 或 Token Account 都需要支付 SOL 作为 rent（约 0.002 SOL）
  - ERC20 没有这个概念，存储成本体现在 gas 里

  ---
  5. 权限体系

  ┌──────────┬────────────────────────────────┬────────────────────┐
  │   权限   │           SPL Token            │       ERC20        │
  ├──────────┼────────────────────────────────┼────────────────────┤
  │ 铸币权   │ mint_authority（可撤销/转移）  │ 合约 owner 自定义  │
  ├──────────┼────────────────────────────────┼────────────────────┤
  │ 冻结权   │ freeze_authority（可冻结账户） │ 无标准，需自己实现 │
  ├──────────┼────────────────────────────────┼────────────────────┤
  │ 关闭账户 │ close_account 可回收 rent      │ 无此概念           │
  └──────────┴────────────────────────────────┴────────────────────┘

  ---
  6. Token-2022（扩展版 SPL Token）

  Solana 推出了新版 Token-2022 程序，原生支持：
  - Transfer Hook（转账时执行自定义逻辑）
  - Confidential Transfer（隐私转账）
  - Transfer Fee（链上手续费）
  - Permanent Delegate（永久委托）

  这些在 ERC20 里需要自己写合约扩展（如 ERC20Fee、ERC777 等）。

  ---
  一句话总结

  ▎ ERC20 = 一个代币一个合约，状态集中在合约 storage
  ▎ SPL Token = 所有代币共用一个程序，状态分散在各自的链上账户（Mint + Token 
  Accounts）

⏺ SPL Token 深度解析

  账户体系（最核心）

  SPL Token 涉及三种账户，搞清楚它们的关系是理解一切的基础：

  ┌─────────────────────────────────────────────────────┐
  │                    spl-token 程序                    │
  │         TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss4        │
  └──────────────┬──────────────────────────────────────┘
                 │ 管理
                 ▼
  ┌─────────────────────────┐
  │      Mint Account        │  ← 代币本身的定义
  │  - supply (总供应量)      │
  │  - decimals              │
  │  - mint_authority        │  ← 谁能增发
  │  - freeze_authority      │  ← 谁能冻结
  └──────────┬──────────────┘
             │ 1:N
             ▼
  ┌─────────────────────────┐
  │     Token Account        │  ← 某个用户持有该代币的"口袋"
  │  - mint (属于哪个代币)    │
  │  - owner (谁控制这个账户) │
  │  - amount (余额)         │
  │  - delegate              │
  │  - state (frozen?)       │
  └─────────────────────────┘

  ---
  Mint Account 详解

  地址: 任意公钥（创建时生成）
  大小: 82 bytes（固定）
  数据:
    mint_authority: Option<Pubkey>   // 增发权限，None = 永久锁死供应量
    supply:         u64              // 当前流通总量
    decimals:       u8               // 精度，如 USDC = 6
    is_initialized: bool
    freeze_authority: Option<Pubkey> // 冻结权限

  创建 Mint 的本质是：
  1. 创建一个普通账户（分配 82 bytes 空间，支付 rent ~0.0015 SOL）
  2. 调用 spl-token 的 InitializeMint 指令初始化它

  ---
  Token Account 详解

  地址: 任意公钥（通常用 ATA 派生）
  大小: 165 bytes（固定）
  数据:
    mint:                 Pubkey       // 对应哪个 Mint
    owner:                Pubkey       // 谁有控制权
    amount:               u64          // 余额（注意是整数，要除以 10^decimals）
    delegate:             Option<Pubkey>
    state:                AccountState // Initialized / Frozen / Uninitialized
    is_native:            Option<u64>  // 是否是 Wrapped SOL
    delegated_amount:     u64
    close_authority:      Option<Pubkey>

  ---
  ATA（Associated Token Account）

  每个用户对每种代币只需要一个"标准口袋"，ATA 就是这个标准口袋：

  ATA 地址 = PDA(
      seeds = [owner_pubkey, token_program_id, mint_pubkey],
      program = associated_token_program
  )

  特点：
  - 地址完全确定性派生，任何人都能算出你对某个代币的 ATA 地址
  - 不存在时需要创建（付 rent ~0.002 SOL）
  - 可以 close_account 回收 rent（余额清零后）

  用户 Alice (wallet: AAAx...)
    ├── USDC ATA:  PDA(AAAx... + USDC mint)  → amount: 100_000000
    ├── SOL ATA:   native wallet 本身
    └── JUP ATA:   PDA(AAAx... + JUP mint)   → amount: 50_000000000

  ---
  核心指令一览

  InitializeMint       创建代币（设置 decimals, authority）
  InitializeAccount    创建 Token Account
  MintTo               增发代币到某个 Token Account  [需要 mint_authority]
  Transfer             转账（从一个 Token Account → 另一个）
  Burn                 销毁代币
  Approve              授权 delegate 可花费一定数量
  Revoke               撤销授权
  FreezeAccount        冻结账户（被冻结后无法转账）[需要 freeze_authority]
  ThawAccount          解冻账户
  CloseAccount         关闭 Token Account，回收 rent SOL
  SetAuthority         转移或撤销 mint/freeze authority

  ---
  和 ERC20 授权的差异

  ERC20：
  approve(spender, amount)   // 授权
  transferFrom(from, to, amount) // 被授权方执行

  SPL Token：
  Approve(delegate, amount)  // 授权某个 delegate 最多花 amount
  Transfer(...)              // delegate 直接调用 Transfer，source 是被授权账户

  关键区别：SPL Token 的 delegate 直接操作 Token Account，不需要 transferFrom
  这个单独指令。

  ---
  Wrapped SOL（wSOL）

  SOL 不是 SPL Token，但可以包装成 wSOL 参与 DeFi：

  wSOL Mint = So11111111111111111111111111111111111111112（固定地址）

  创建 wSOL Token Account → 往里转 SOL → amount 就等于 lamports
  关闭 wSOL Token Account → SOL 返回 owner

  这就是为什么 Jupiter/Raydium 交易时有时会看到"create wSOL account"的操作。

  ---
  Token-2022 扩展（新标准）

  Token-2022 是升级版，程序地址不同：TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb

  新增扩展（按需选择）：

  ┌─────────────────────────────────┬──────────────────────────────┐
  │              扩展               │             功能             │
  ├─────────────────────────────────┼──────────────────────────────┤
  │ TransferFee                     │ 每次转账自动扣手续费给协议   │
  ├─────────────────────────────────┼──────────────────────────────┤
  │ TransferHook                    │ 转账时触发自定义程序逻辑     │
  ├─────────────────────────────────┼──────────────────────────────┤
  │ ConfidentialTransfer            │ 隐私转账（金额加密）         │
  ├─────────────────────────────────┼──────────────────────────────┤
  │ PermanentDelegate               │ 永久委托，可无限花费任何账户 │
  ├─────────────────────────────────┼──────────────────────────────┤
  │ MintCloseAuthority              │ 允许关闭 Mint Account        │
  ├─────────────────────────────────┼──────────────────────────────┤
  │ NonTransferable                 │ 灵魂绑定代币（SBT）          │
  ├─────────────────────────────────┼──────────────────────────────┤
  │ MetadataPointer + TokenMetadata │ 链上元数据，无需 Metaplex    │
  └─────────────────────────────────┴──────────────────────────────┘

  ---
  实际创建 SPL Token 的流程

  // 1. 创建 Mint Account
  const mint = await createMint(
      connection,
      payer,           // 付 rent 的人
      mintAuthority,   // 增发权限
      freezeAuthority, // 冻结权限（null = 无冻结功能）
      decimals         // 精度
  );

  // 2. 为用户创建 ATA
  const ata = await getOrCreateAssociatedTokenAccount(
      connection,
      payer,
      mint,
      userWallet.publicKey
  );

  // 3. 增发
  await mintTo(
      connection,
      payer,
      mint,
      ata.address,    // 目标 Token Account
      mintAuthority,
      1000 * 10**decimals
  );

  // 4. 转账
  await transfer(
      connection,
      payer,
      sourceATA,
      destinationATA,
      owner,
      100 * 10**decimals
  );

  ---
  关键要点回顾

  1. Mint Account  → 代币定义（唯一）
  2. Token Account → 用户余额（每人每币一个）
  3. ATA           → 标准化的 Token Account 地址派生方式
  4. Rent          → 链上存储需付押金，关闭账户可回收
  5. Authority     → mint/freeze 可转移，可撤销（撤销后不可逆）
